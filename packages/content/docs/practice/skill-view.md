skill 并不是灵丹妙药，他和 MCP 一样都有缺点，而且我认为现在有滥用的嫌疑

首先是优质的 skill 很少，从最近爆火的 openclaw 就能看出来，他里面的 skill 很明显就是纯 AI 生成的，这不是说 AI 生成的 skill 不好，经过良好验证的 skill 是没有问题的，但我看里面的内容，我很怀疑这些 skill 的可用性。

还有就是普适性，skill 一般会有 scripts 来与外界进行复杂交互，但是用户很有可能没有对应的代码的运行时，即使有也有可能没有 script 依赖的包，这点和 MCP 很像，也是非常大的用户痛点。同时对开发者来说，skill 的版本管理更为困难，反而不如 MCP 标准考虑的周全

我认为 skill 的毛病还有很多，比如不适合分发，感觉它更像是一个“私人”的东西，而非具有确定性的公共产品，如此种种，难以尽数。但我也承认，他确实有效，毕竟 skill 就像是零散打包的说明书 + 工具，这些东西给人看也可以做到同样的效果

这里并不是有意要黑 skill，只是看到 openclaw 的爆火，心里总感觉不太对劲。这是我首次发表上面这些“暴论”，希望时间和现实能评判上面的思考，也给予我更多的经历与反思
